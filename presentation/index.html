Analyst + Critic: Contested Diagnosis

Analyst

  • 3 ranked hypotheses with confidence scores
  • Each hypothesis cites supporting + contradicting evidence IDs
  • Deterministic heuristics or OpenAI Responses API (strict JSON)
postgres_down output
  1. PostgreSQL dependency outage 0.93
  2. Connection pool exhaustion 0.46
  3. Recent deploy regression 0.18

Critic

  • Reviews hypotheses against the same evidence
  • Emits findings, hallucination risks, missing data
  • Enforces safety notes before planner acts
What Critic catches
  • Unsupported causal leaps
  • Missing corroborating signals
  • Approval boundary violations
If Redis is up but basketservice logs show Redis errors, Analyst says "path degraded" not "outage". Critic would flag if Analyst overclaimed.

Planner: Actionable Incident Brief

Incident Brief: postgres_down

PostgreSQL dependency outage. Evidence spans logs, metrics, traces, and runbook guidance for catalogservice.

Auto-safe

  1. Inspect PostgreSQL health, restart history, readiness probes
  2. Correlate failing traces with first DB connection-refused logs
  3. Validate recovery via 5xx rate and latency after mitigation

Needs Approval

  1. Restart or fail over PostgreSQL if confirmed down
  2. Roll back DB connectivity config if recent change implicated
Open questions: No unresolved data gaps in the mock scenario.
This is where the system becomes useful to an operator. Diagnosis alone is not enough; the user needs next actions with explicit safety boundaries.

LLM comparison: 100% top-1 accuracy but 75% qualitative score -- model plans use different phrasing and sometimes omit safety boundaries. Deterministic mode is the reliable evaluation baseline.
The qualitative checks justify multi-agent architecture. A single-agent summarizer could match top-1 accuracy but would not naturally produce evidence-linked citations, explicit safety boundaries, or internally contested hypotheses.

Strengths, Limits, and Future Work

Strengths

  • 4 agents with explicit contracts
  • Full reasoning traces per incident
  • Quantitative + qualitative eval harness
  • Safety boundaries on all actions
  • Reproducible local demo with real telemetry

Current Limits

  • 2 scenarios with ground-truth labels
  • Retriever is deterministic, not model-driven
  • No true service dependency graph
  • LLM mode less structurally predictable
  • Live demos subject to latency noise

Future Work

More scenarios (frontend overload, cascading failures) • True dependency graph • Model-driven retriever • Visual dashboard for traces

"The multi-agent pipeline turns evidence into safer, traceable incident briefs."

End on credibility, not hype. The system is strongest as a traceable incident-analysis assistant, not a fully autonomous fixer.
