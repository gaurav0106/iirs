## Local-only changes made during e2e debugging

These are not tracked by git because they live under `.external/aspire-samples/...`, but they matter if you resume this later.

### 1. Frontend internal service calls were switched to HTTP by default

File:

- `.external/aspire-samples/samples/aspire-shop/AspireShop.Frontend/Program.cs`

Change:

- Internal calls now default to:
  - `http://catalogservice`
  - `http://basketservice`
- Override back to HTTPS with:
  - `IIRS_ASPIRE_INTERNAL_SCHEME=https`

Why:

- Linux dev-cert trust was brittle enough that internal HTTPS between spawned Aspire resources kept failing with `UntrustedRoot`.

### 2. AppHost was patched to forward OTLP env vars into child services

File:

- `.external/aspire-samples/samples/aspire-shop/AspireShop.AppHost/AppHost.cs`

Change:

- If `IIRS_OTLP_ENDPOINT` is set, AppHost now injects:
  - `OTEL_EXPORTER_OTLP_ENDPOINT`
  - `OTEL_EXPORTER_OTLP_PROTOCOL`
  - `OTEL_EXPORTER_OTLP_HEADERS=""`
- This is applied to:
  - `catalogdbmanager`
  - `catalogservice`
  - `basketservice`
  - `frontend`

Why:

- Without this, the Aspire child processes exported OTLP to Aspire's own local endpoint (`https://localhost:16224`) instead of the local LGTM stack. `verify-live` then saw no application telemetry at all.



```bash

```

```bash

done
```


```bash

```
